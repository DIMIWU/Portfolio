Project Overview

有讲课堂是国内一家互联网在线教育平台，为大众提供全面的技能学习入口。作为传统教育的补充，平台课程覆盖100多个类目，有讲课堂将自己定位为一所社会技能大学。随着知识付费2.0时代的到来，市场上也涌入大量的产品提供资源服务，为教育市场助力的同时，处处伴随着机遇和挑战。
2021年我们对有讲课堂App做了小幅的改版。重新梳理首页页面结构与feed流，优化社区模块，优化产品推荐机制。加强品牌意识，重新设计了品牌logo与相关的品牌印象图标,统一设计语言。

发现问题

在设计前期，我们对三位有讲课堂的真实用户进行深度访谈，主要方向是以用户的基本信息与学习条件，对平台的使用体验，满意与不满意的地方。综合访谈结果，并提取有利于产品发展思考的问题。

一．用户粘性低，在使用的过程中对产品产生不确定感。

一些热门的文章即使有很多用户阅读，但讨论度却不高。在App中，在多处出现相同的文章，重复性较高，用户难以区分模块的不同。

用户反馈：“在学习的时候大多是利用下班的时间断断续续的学习，不知道自己学的怎么样了。忙着忙着就会忘记我还在学这些课程。”

付费学习的用户不能直观的知道自己的学习进度与状况，没有及时得到反馈，学习过程中的不确定性让用户粘度大大降低，用户流失严重。

二．产品重点不突出，用户转换率低

用户反馈：“能看到一些不错的职场类文章，但是不是很明确什么样的课程能帮助自己能够有更好的提升。“

有讲课堂不是一个高频的软件，在使用过程中，更容易使用之前的使用过的路径。在用户访谈的过程中发现，有讲课堂有着不错的内容，但用户想要深入的学习，很难找到相关的课程，产品转化节点不足。

三．品牌印象不深刻

用户反馈：“感觉有讲课堂不像是一个职业教育类的App，不够有趣。”

在这个问题上，我们需要从两个层面上深入思考。
在产品层面上，即使在平台上有许多不错的文章，但是课程的引导性不强，转化率低。用户难以识别不同性质模块的区别，也很难让有兴趣的客户知道我们有相关的课程可以学习。
从视觉层面上，有讲课堂的整体色彩明度较低，色彩不够丰富，很难刺激用户的积极性。
来自用户的反馈：“慢慢的好像失去了打开学习的动力了。”

产品目标

针对以上问题，确立了接下来改版的三个主要目标

1．提升品牌感知，加深品牌印象

生成品牌的超级符号，建立深刻的品牌感知

2．增强用户活跃度，增强产品黏性

增强用户对学习讨论度，鼓励用户在学习的同时也对外输出，并获得成就感

3．激发用户付费率，突出产品重点与转化节点

增加用户了解课程的渠道，并增加节点快速转化

提升品牌感知力

KR1:强化品牌印象

Action 1：优化品牌Logo

用户反馈：
“感觉有讲课堂不像是一个职业教育类的App，在这里很难发现什么有趣能吸引我的课程”
“慢慢的好像失去了打开学习的动力了。”

从视觉层面上，有讲课堂的整体色彩丰富度较低，色彩不够丰富，很难刺激用户的积极性。视觉层面缺少对品牌元素的提炼，和统一的透传方式。
品牌图标充满活力，但因为设计没有更新，图标风格过时，且品牌感知力不强。 在使用中，图形本身在认知层面上存在认知层面的成本，有讲课堂作为一个职业教育类平台，不容易让用户在使用过程中对其产生深刻的印象，并对其有明确的产品定位。
在本次的设计改版中，我们诉求产品能过让用户在使用过程中有潜移默化的感知，对产品有更明确的定位感。

Logo图形分析

形状和色彩一样，是传达视觉感受的重要手段。有讲课堂的原始品牌Logo使用了倾斜的平行四边形作为基础形状。从心理学角度来说，给人一种活力，上进的感觉，但与此同时，这样不稳定的形状也意味着一种不安全感。有讲课堂作为一个职业技能学习平台，更希望向用户传递一种可信赖的感受。

根据形状心理学，我们在新一代的品牌图标上，选择了相对稳定的矩形，一种我们生活中最常见的图形作为基础图形，给人带来亲切，可靠，安定的感觉。但太过于安定却又缺乏了变化，所以在设计中，我在矩形的基础上，给字体添加了一点弧度，让图标线条显得更加柔顺，添加亲和力。

在设计的过程中发现，有讲课堂作为教育类产品，从视觉感知程度上分析，难以给用户有直接的表意。因此，我对产品的品牌图标再次进行重新升级。在画草图的过程中，参考竞品的图标，提取教育的元素。

在多次尝试后，我们做出了当下的最优选择。我们将第一代的Logo进行改版迭代，在第一代“有讲”的字体设计的基础上融入书本的元素，从产品上，让老用户对新的产品设计不会有太大的疏离感。为了突出产品的重点与特色，结合了书本与等边六边形稳定却带有活力的感知，极具爆发力。从形状和表意上也能达到目的。

Action 2：定义品牌超级符号

书本形状的等边六边形认知成本低，符合品牌调性。书本形状的六边形与学习之间存在的强关联，将形状融入产品的细节中，品牌感能够快速得到强化，提升产品的记忆度。

KR2:统一设计语言

Action 1：规范图标设计

重新定义App内所使用的图标风格及规范

Action 2：统一色彩系统

在色彩的选择上，主色依旧沿用原来的橙色。但原有的以渐变色为主，在视觉感受上略显陈旧。我们产品的主要人群为90后，鲜明简洁的色彩更能符合当代年轻人的喜好。在新版的颜色选用了明度与饱和度相对较高的颜色。

Action 3：建立文字规范

定义iOS和Android两大平台的中文/英文/数字所用的字体/字重和字号，规范其使用场景。

增强用户活跃度

KR1:重新架构社区模块

有讲课堂在1.0版本中以PGC为主，包含了许多专业性的文章。广场模块与App首页内容高度重合。我们已经有了好的内容，想要建立好的社区生态，就需要建立一个良好的链接.

Action 1：改善热门话题版块

问题：在旧版的广场页头部为热议话题，但是在用户访谈的过程中发现，旧版的热议话题相对信息比较混乱，层次拉不开，难以吸引用户。

针对这个问题，我提出了三个方案：

方案1:
在1.0版本的话题板块中，采用向左滑动的方式。第一个热门帖子成为能否吸引用户，提高点击率成为关键要素。在新版的设计中，我们将当下最火热的一个话题信息更详细的呈现，丰富画面色彩，从而吸引用户的眼球。方案1呈现的热门话题少，但是由系统推送的当下最具热门的帖子，精准了内容，信息也更加丰富。

方案2:
使用当下头部社交类Apps常使用的列表型呈现方式，如抖音，微博等，减少了用户的使用成本，符合用户体验。

方案3:
讲热门话题分类，根据用户的喜欢的话题进行分类.

因为有讲App是一个教育类的平台，用户群体相对于泛知识类App较小。推送最热门的话题，有利于增加话题的讨论度，从而让用户逐渐在社群中产生归属感。方案1的设计由于热门帖子承载更多的信息，也更具有活力。最终我选用方案1。

Action 2：建立热门学习小组

上瘾模型

提到上瘾，我们总是想到游戏或者是抖音等App。Hook模型是让用户养成习惯的行为模型。热门小组是2.0版本后新增的功能板块，属于一个半封闭式的交流圈，经过申请方可进入。 用户找到自己感兴趣的小组，从而触发让用户加入。找到感兴趣的“同类人”更能引起共同话题，并得到同伴的回应或认同，从而在平台中投入更多的时间。

半封闭式的小组给成员带来更强烈的归属感，更有利于产生有价值的内容，从而产生用户黏性。用一种具有仪式感的行为促进群体团结，提升了社区的属性和氛围浓度。

在此，针对热门小组的设计我同样也提出了三个方案：
方案1:使用左图右文的方式，并显示小组的最基本信息。
方案2:使用卡片的形式。
方案3:（使用方案）使用向左滑动的小卡片形式，只显示小组图像与名称。

方案1与方案2屏幕占比率相同。从美观的角度考虑，方案2的丰富度是最强的，更能吸引用户。但太过跳脱，影响我们想带给用户的节奏感。在最终的方案里，我选择了方案3，此方案在页面中占比较小，为第三部分的高潮做出了基垫。

Action 3：优化推荐热帖与关注列表

广场作为社区属性的模块，与首页的文章推荐模块内容高度重合，以PGC为主，两个模块出现相同的文章比率高，让用户产生混乱感。

一个好的社区需要具备好的内容和优质的关系链。有讲课堂具备了许多优质的专业性文章，但关系链薄弱。UGC创作渗透率以及用户互动的渗透率越高，社区属性越强，用户对产品的黏性越高。

用户反馈：“感觉有讲课堂不像是一个职业教育类的App，在这里很难发现什么有趣能吸引我的课程”

在产品层面上，即使在平台上有许多不错的文章，但是课程的引导性不强，转化率低。用户难以识别不同性质模块的区别，也很难让有兴趣的客户知道我们有相关的课程可以学习。

一些热门的文章即使有很多用户阅读，但讨论度却不高。相关推荐课程“含蓄”在App中，在多处出现相同的文章，重复性较高，用户难以区分模块的不同。

间接竞品：微博
微博是一种以关系链主，内容为辅的混合型社区。它不仅仅可以跟我们熟知的人可以有所联系，更能与官方，明星等直接建立连接。这是一种多维度关系链。多维度的链接产生多维度内容，能给社区带来强有力且稳定的关系。

直接竞品：网易公开课
网易公开课作为有讲课堂的直接竞品，广场版块以热门的帖子为主。

针对以上对竞品的分析，我设计了多种社区帖子的展示形式，以现在社交媒体现有的上文下图形式，以减少用户的使用成本。在设计中穿插不同的形式，避免了视觉疲劳。

KR2:巩固用户学习体验

用户反馈：
“在学习的时候大多是利用下班的时间断断续续的学习，不知道自己学的怎么样了。忙着忙着就会忘记我还在学这些课程。”

经过调查与用户反馈发现，大多数用户以碎片化学习方式为主，如上下班的地铁里，或者是中午的午休时间。这种零碎的学习方式难以让用户在大脑中形成体系，甚至经常忘记自己的课程，用户黏性大大降低。

Action 1：优化学习课程表

付费学习的用户不能直观的知道自己的学习进度与状况，没有及时得到反馈，学习过程中的不确定性让用户粘度大大降低，用户流失严重。

想让用户更加积极参与课程，需从内在触发用户对学习和产品价值的认同感，达到自我成就。

Action 2：升级学习报告

为什么需要学习报告

学员每天学习的时间相对比较零碎，零碎的时间或者是学员沉浸式的学习方式，对自身的学习状况没有直观的认知。
教学层面上，教师与学生能够正对自身的状况对学习进行随时的调整。

将Hook Model融入到学习报告中

平台为客户提供服务，学员查看报告，提供学员的平均数据从而产生竞争心理而去学习。在学习的过程中，获得成就感后并养成习惯，对产品产生依赖。

激发用户付费欲

KR1:增加社区付费转化点

Action 1：增加文章与热帖付费点

直接竞品：CCTalk
CCTalk的首页也有学员评价的模块，学员在评论课程的同时也附上了课程购买的链接，学员的推荐更有说服力，更有利于提高了产品的转化率与复费率。

间接竞品：微博
微博在帖子中添加了快速点击购买的链接，快速转化客户的复费率。

根据竞品分析，我们在广场热帖模块添加了课程链接，以便快速转化用户。

KR2:突出产品重点

Action 1：重构页面占比率

在有讲课堂的框架布局中，主要分为了三个部分，头部banner与搜索、金刚区、直播列表和文章。首页作为用户最常用的页面，旧版的屏效低，产品重点不突出。

从竞品分析中，我们分析了腾讯课堂、CCTalk和网易云课堂，他们的产品一目了然，课程也更容易在首页找到。因此我们对首页的架构进行了重新整理。

Action 2：优化品类区与底部导航栏

由于对整体结构的重新调整，我们也对品类区做了竞品分析。

“中国大学慕课”、“腾讯课堂”、“CC Talk”与“知鸟”将品类区将课程种类做了细分，大大提高了用户寻找课程的精准度。 “腾讯课堂”与“CCTalk”以扁平风格为主，色彩明度与饱和度较高，设计风格简单，色彩丰富，更加符合当下年轻人的风格。

跟据竞品分析，我们确定了品类区的设计风格，以当下流行的高明度的扁平icon为主。品类区一共分为十个类别。

在本次的设计改版中，我们也优化了底部导航栏的设计。

Action 3：优化课堂直播列表

在旧版的设计中，直播模块以卡片翻页的形式呈现，屏幕占比较大，呈现信息有限，也增加了用户的使用成本。大大降低了用户的点击率。

在新的改版中，为了提高屏幕效率，直播模块也做了改版。我同样提出了三种方案

方案1：采用了左文右图的形式，符合现在大多数App中的阅读方式，降低用户使用成本。
方案2：采用左图右文的方式，图片在左侧，有趣的图片更容易引起用户的兴趣。
方案3：采用卡片向左滑动的形式，大大增加了屏幕效率。

依据尼尔森F阅读模型，用户习惯从左到右，至上而下的阅读方式，方案1将文字置于前方，更有利于用户快速掌握课程信息。方案2从视觉层面看，右侧会形成很大的缺角，且配图为教师照片，信息颗粒度低。方案3大大增加了屏效，但增加了用户的使用成本。所以我选择了方案1。

Action 4：建立热门课程排行

添加热门课程排行版让用户更容易选择课程。

方案1的排行榜更加细致，提供内容更加丰富。但方案2更加符合整体的风格，制定的模版与颜色使用更符合本次改版的风格。

Action 5：重建文章与视频列表

在旧版大的设计中，专题模块呈现占比较大，但课程转化率差。专题版块以PGC为主，输出内容专业，以文章与视频为主。竞品分析中，腾讯课堂与CCTAlk分别使用了瀑布流和列表的方式。
